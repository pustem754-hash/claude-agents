# content — генерация постов под соцсети

Фокус-скилл: один проход «тема/источник → готовые посты под выбранные платформы».
Лимиты, тон и форматы платформ НЕ держим в голове — они в `tools/platform-specs.json`
(детерминированный референс). Ресёрч и транскрибацию делегирует вызывающий.

## Когда брать / не брать

- ✅ «Напиши пост для LinkedIn», «адаптируй текст под IG и Threads», «контент-план на неделю».
- ❌ «Сделай лендинг» → `site`. «Найди факты по теме» → `deep-research`.
  «Разбери это YouTube-видео» → `transcript-analyst` / `youtube-analyzer`.

## Процесс (плейбук)

1. **Бриф.** Собери: тема, платформа(ы), аудитория, тон (экспертный/дружеский/
   провокационный), желаемый CTA. Чего нет — выбери разумный дефолт и зафиксируй.
2. **Источник.** Если дан исходник (статья, транскрипт, отчёт ресёрча, документ) —
   вытащи 2-5 ключевых тезисов. Если источника нет — пиши экспертный контент по теме.
3. **Лимиты — из конфига.** Открой `tools/platform-specs.json`, возьми для каждой
   целевой платформы: `char_limit`, `optimal_length`, `hashtags`, `tone`, `format`.
   Не выдумывай лимиты по памяти — это сквозной принцип «справочник, не угадайка».
4. **Каркас.** Строй каждый пост по схеме **hook → тело (2-5 тезисов) → CTA**:
   - *hook* — первая строка цепляет: вопрос, цифра, противоречие, личная история;
   - *тело* — раскрытие тезисов, без воды, каждая строка несёт ценность;
   - *CTA* — конкретный (подпишись / задай вопрос / перешли коллеге), не «лайк если согласен».
5. **Адаптация под платформу.** Под каждую соцсеть — ОТДЕЛЬНАЯ версия, не копия одного
   текста. Уважай `format` (для TikTok — сценарий с таймкодами hook в первые 3 сек;
   для X — тред из постов по `char_limit`; для Telegram — Markdown с абзацами).
6. **Хэштеги и эмодзи.** Кол-во хэштегов — из `hashtags` платформы. Эмодзи — визуальные
   якоря, 3-7 на пост максимум. Не перегружай.
7. **Варианты.** Дай 2-3 варианта поста на выбор (минимум для hook/заголовка).
8. **Отдай.** Сохрани в `agent-runtime/outputs/content/<topic-slug>-<platform>.md` и
   верни пути. Эталон формата вывода — `examples/post-example.md`.

## Правила

- Соблюдай лимиты символов из конфига — это hard-constraint, не рекомендация.
- Никаких штампов («в современном мире», «давайте поговорим о…»).
- Для Threads/X — короткие предложения, без сложноподчинённых.
- Для Threads — ссылки в теле слабые, выноси CTA-ссылку отдельно/в профиль.
- Не выдумывай факты из источника — только то, что реально в нём есть.

## Слои этого скилла

- **tools/platform-specs.json** — справочник платформ: лимиты, длина, хэштеги, тон,
  формат (слой 3, детерминированный конфиг — модель не помнит лимиты).
- **examples/post-example.md** — few-shot эталон: один пост под 3 платформы.
