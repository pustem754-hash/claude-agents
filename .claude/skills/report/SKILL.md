# report — генератор PDF/XLSX отчётов из данных

Фокус-скилл: «готовые данные → файл-отчёт». Сбор данных не входит — это parser /
deep-research / freelance-digest. Движки рендера детерминированы (Правило «код-кодом»):
их НЕ переписывают под каждую задачу, данные идут через JSON-спеку.

## Когда брать / не брать

- ✅ «Собери PDF по топ-10 агентств», «выгрузи этот список в Excel с подсветкой роста».
- ❌ «Найди данные / спарси сайт» → parser/deep-research. «Сделай презентацию» → pptx.
  «Нужен .docx» → docx.

## Процесс (плейбук)

1. **Собери данные в JSON-спеку** по контракту из `tools/report_schema.md`.
   Эталоны: `examples/pdf-spec-example.json`, `examples/xlsx-spec-example.json`.
   Спеку клади в `agent-runtime/shared/` или `tmp/`.
2. **Выбери движок:**
   - Документ с обложкой / карточками / источниками → **PDF**:
     ```
     python .claude/skills/report/tools/build_report_pdf.py <spec.json> outputs/<name>.pdf
     ```
   - Таблица с числами / форматированием / фильтрами → **XLSX**:
     ```
     python .claude/skills/report/tools/build_report_xlsx.py <spec.json> outputs/<name>.xlsx
     ```
3. **Отдай путь** к файлу. Если канал Telegram — отправь файлом.

## Что умеют движки

- **PDF** (`build_report_pdf.py`): обложка с цветными полосами, intro, сводная таблица,
  карточки с полями, наблюдения, источники, футер со стр. Кириллица (Arial). Все секции
  кроме `title` опциональны — нет ключа → секция пропускается.
- **XLSX** (`build_report_xlsx.py`): титул, источник, заголовки, данные, number_format и
  выравнивание по колонкам, условное форматирование `sign` (зелёный >0 / красный <0),
  цветовые шкалы, лист Summary, autofilter, freeze panes.

## Правила

- Данные НИКОГДА не хардкодь в скриптах — только в JSON-спеке. Это убивает антипаттерн
  «новый билдер под каждый отчёт» (раньше в thoughts/ было 3 PDF + 3 XLSX билдера).
- Меняешь оформление (цвета/шрифты/стили) → правишь движок ОДИН раз, не форкаешь скрипт.
- Перед запуском проверь, что JSON валиден и поля соответствуют `report_schema.md`.

## Слои этого скилла

- **tools/build_report_pdf.py** — движок PDF (слой 3, код).
- **tools/build_report_xlsx.py** — движок XLSX (слой 3, код).
- **tools/report_schema.md** — контракт обеих JSON-схем (слой 3, документация).
- **examples/pdf-spec-example.json**, **examples/xlsx-spec-example.json** — few-shot спеки.
